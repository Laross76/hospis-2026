# 1. Информация:
- Проект: Мобильное приложение "Мобильный хоспис"
- Версия приложения: v. 1.0.0
- Дата тестирования: 16.07.2026
- Тестировщик: Россихина Светлана
- Тип тестирования: Функциональное, UX/UI, кроссплатформенное (Android)

# 2. Введение и цели 
Проведино тестирование охватывающие ключевые функциональные возможности приложения.

# 3. Окружение и устройства
- Платформа: Android
- Версия ОС: Windows 11
- Эмулятор: Pixel 4 XL API 36.0

# 4. Результаты тестирования
- Написано 27 тест-кейсов, 125 тестов.
- Автоматизировано 27 тест-кейсов,  59 авто-тестов.
- Все 59 авто-тестов были успешно завершины.
- Составлено три баг-репорта, описывающие обнаруженные ошибки и недочеты в работе приложения.


  Баг-репорт	                 Приоритет	                                                    Описание
    # 1	                          Высокий	             Недоступна страница "Политика конфиденциальности" при попытке перейти по ссылке
                                                       Эта ошибка препятствует доступу к важной информации о политике конфиденциальности компании, что  негативно скажется на доверии пользователей к сайту. 
    # 2	                          Высокий	             Недоступна страница при переходе по ссылке 'Terms os use' из раздела о приложении(About)
                                                       Ошибка может привести к потере доверия со стороны клиентов и потенциальных юридических последствий, связанных с несоблюдением требований к защите данных.
    # 3	                          Высокий	             Сброс введенных данных при смене ориентации экрана на странице авторизации(Authorization)
                                                       Почти заполненные данные (например, сложный пароль) стираются, заставляя пользователя вводить всё заново. Часть пользователей просто закроет приложение или сайт, устав повторно заполнять форму.

- Время затраченное на проверку в ручную  - 3 часа 30 минут
- Врямя затраченное на автоматизированную проверку - 35 минут

# Заключение
По результатам проведенного тестирования мобильного приложения "Мобильный хоспис" (версия v.1.0.0) была выполнена комплексная проверка функциональности, UX/UI и кроссплатформенной совместимости на устройствах под управлением Android (эмулятор Pixel 4 XL API 36.0).

В общей сложности было разработано и выполнено 125 тестов, из которых 59 были автоматизированы с использованием Allure. Все автоматизированные тесты завершились успешно, что подтверждает корректность реализации основного функционала приложения.

Однако были выявлены критические дефекты, препятствующие полноценной работе приложения:

Недоступность страниц "Privacy Policy" и "Terms of Use" из раздела "About", что нарушает требования к информационной прозрачности и может негативно повлиять на доверие пользователей.
Потеря данных при изменении ориентации экрана на странице авторизации, что создает неудобства для пользователей и увеличивает риск их оттока.
Сравнительный анализ времени проверки показал значительное преимущество автоматизации: 35 минут против 3 часов 30 минут при ручном тестировании, что демонстрирует высокую эффективность внедренного подхода.
